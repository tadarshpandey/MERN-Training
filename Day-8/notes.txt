-->> Synchronous vs Asynchronous Programming in JavaScript:

1. Synchronous Code:
- Executes line by line, sequentially from top to bottom.
- Each operation must wait for the previous one to finish before starting.
- It is "blocking" (e.g., a heavy task will freeze the program until it completes).

Example:
console.log("1. Start");
console.log("2. Middle");
console.log("3. End");
// Output strictly follows order: 1. Start -> 2. Middle -> 3. End


2. Asynchronous Code:
- Allows the program to continue executing the rest of the code while waiting for a long-running operation to finish (e.g., fetching data from an API, reading files, timers).
- It is "non-blocking".
- JavaScript handles async tasks using Callbacks, Promises, or Async/Await.

Example:
console.log("1. Start");

// setTimeout is an asynchronous web API function
setTimeout(() => {
    console.log("2. Middle (Async Task Done!)");
}, 2000); // Wait for 2 seconds (2000ms) but don't block the rest of the code

console.log("3. End");
// Output: 
// 1. Start 
// 3. End 
// (Wait 2 seconds...) 
// 2. Middle (Async Task Done!)
